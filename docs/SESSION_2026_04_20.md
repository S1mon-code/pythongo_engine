## 🔑 关键发现(from 源码审计)

### Pythongo API 的隐含约定

1. **`trade.direction`** 是 `Literal["0", "1"]`(数字字符串),**不是**中文"买"/"卖"
2. **`OrderData` 没有 `.volume`** — 只有 `total_volume / traded_volume / cancel_volume`
3. **`get_position()` 永远不返 None** — 返回空 `Position()` 对象(`net_position = 0`)
4. **`Scheduler("PythonGO")` 是全局单例** — `on_stop` 会清所有 jobs(策略自己的定时器要用不同 name)
5. **`make_order_req` 在 `self.trading=False` 时静默 return None** — 我们 `if oid is not None` 检查是对的
6. **`on_order` 根据中文 status 分发 `on_order_cancel / on_order_trade`** — override 必须调 `super().on_order(order)`
7. **`on_error` 默认有 0004 自动流控** — override 没调 super = 失去流控(本次修复补回)
8. **`producer.close[-1]` 在 `real_time_callback` 时是未完成 bar 实时价** — 不要在里面做信号计算
9. **`TickData.last_volume` 是 class-level dict 运行时算的 delta** — 不是 broker 字段
10. **`auto_close_position` SHFE/INE 可能一次发 2 单**(先平昨再平今),只返回最后一个 order_id
