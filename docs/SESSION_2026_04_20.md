## 🔑 关键发现(from 源码审计)

### Pythongo API 的隐含约定

1. **`trade.direction`** 是 `Literal["0", "1"]`(数字字符串),**不是**中文"买"/"卖"
2. **`OrderData` 没有 `.volume`** — 只有 `total_volume / traded_volume / cancel_volume`
3. **`get_position()` 永远不返 None** — 返回空 `Position()` 对象(`net_position = 0`)
4. **`Scheduler("PythonGO")` 是全局单例** — `on_stop` 会清所有 jobs(策略自己的定时器要用不同 name)
5. **`make_order_req` 在 `self.trading=False` 时静默 return None** — 我们 `if oid is not None` 检查是对的
6. **`on_order` 根据中文 status 分发 `on_order_cancel / on_order_trade`** — override 必须调 `super().on_order(order)`
7. **`on_error` 默认有 0004 自动流控** — override 没调 super = 失去流控(本次修复补回)
8. **`producer.close[-1]` 在 `real_time_callback` 时是未完成 bar 实时价** — 不要在里面做信号计算
9. **`TickData.last_volume` 是 class-level dict 运行时算的 delta** — 不是 broker 字段
10. **`auto_close_position` SHFE/INE 可能一次发 2 单**(先平昨再平今),只返回最后一个 order_id

### 确认设计选择正确性

- CLAUDE.md 说「手写 indicator 避免 QBase 信号偏差」**100% 正确** — talib.ADX 不返 PDI/MDI,talib ATR 用 SMA 不是 Wilder,talib donchian 不返 mid
- CLAUDE.md 说「pythongo.ui.BaseStrategy」**正确** — 重写了 on_init/on_start/on_stop 追加 widget 逻辑

## 🔧 跨日生命周期扩展(晚上追加)

Simon 指出每日 **18:00-18:30 无限易清算** + **21:00 前手动重开**的生产场景。
原 AL V8 `_evaluate_startup` 只处理 `target > net_pos`(开/加仓),漏了 3 种:

| 场景 | 旧行为 | 新行为(v2) |
|------|-------|-----------|
| `target > net_pos`(开/加仓) | executor | executor(不变) |
| **`target == 0 + net_pos > 0`**(信号消失) | ❌ return 不平仓 | ✅ **强制平仓** `auto_close_position urgent` |
| **`target < net_pos + target > 0`**(信号减弱) | ❌ return 不减仓 | ✅ **减仓**到 target |
| `target == net_pos`(持仓一致) | return | log "无动作" |

`forecast=0` 时自动 `target=0` 进入平仓分支。Re-entry guard 移到 forecast 判断**之前**,防止 forecast>0 但同 bar 刚止损的 whipsaw。

同步给 `TestAllFixes.py` startup eval 加"若 net_pos > 0 → 强制平仓"测试分支,让 Simon 每日重启时能在 smoke test 看到 `[STARTUP_EVAL] 平仓` log。
