# Session 2026-04-17 — Tick 止损重构 + Scaled Entry 入场重构

## 起因

早上 10:00 AL V8 策略移动止损触发时:
```
10:00:01 close=25375.0 <= 移损线25443.4
```
价格已经低于移损线 **68 点** 才触发,说明止损延迟严重。

根本原因: 止损在 H1 bar 收盘时才判断(`_on_bar` 调 `self._risk.check()`),09:00-10:00 整一个小时价格已经跌破 trail line,系统完全无感。

---

## 解决轨迹(7 次迭代,一天内完成)

### Phase 1: Tick 止损重构 (commit `90def0d`)
**目标**: 止损从 bar 级改为 tick 级硬止损 + M1 级移动止损

**改动**:
- `src/modules/risk.py`:`RiskManager` 新增 3 方法
  - `update_peak_trough_tick(price, net_pos)` — 每 tick 追踪持仓极值
  - `check_hard_stop_tick(price, avg_price, net_pos, pct)` — 每 tick 硬止损(0 延迟)
  - `check_trail_minutely(price, now, net_pos, pct)` — 分钟门控移动止损(降噪)
- `RiskManager` 对称支持做多/做空(net_pos 符号驱动)
- 向后兼容 `check()` 保留,服务 equity/portfolio/daily 止损
- AL V8 率先集成:`_on_tick_stops` + `_exec_stop_at_tick`

**测试**: 23 个 pytest,覆盖 peak/trough 对称、分钟门控、AL 10:00 incident replay

### Phase 4: Scaled Entry Executor (commit `3289213`)
**讨论**: Simon 重新定义建仓需求 — "像人类交易员一样交易"

**核心理念**:
1. 信号 = 本 bar 内仓位目标(不是瞬时打满)
2. 先开底仓(bar 开始 1-2 min),再分批
3. 看多价格越低越买(逆向加仓,VWAP 参考)
4. 止损触发 → 本 bar 锁定放弃剩余
5. 挂单追 bid1(不穿盘口)
6. 反转强烈可超目标(6→8 手,下 bar 回归)
7. 架构原生支持大仓位(未来 100-1000 手)
8. **每次开仓评动态 urgency** — urgency 越高让越多 tick

**引入**:
- `src/modules/rolling_vwap.py` — `RollingVWAP` 滚动 30min 窗口
- `src/modules/execution.py` — `ScaledEntryExecutor` 状态机
  - 5 个状态:IDLE → BOTTOM → OPPORTUNISTIC → FORCE → COMPLETE, 任意 → LOCKED
  - 动态 urgency 评分(0-1 连续):time 40% + deficit 30% + signal 15% + opportunity 15%
  - urgency × 10 = 穿盘口 tick 数

V8 + TestFullModule 集成,测试 114/114 绿

### Phase 5: 审计修复 v2 (commit `458a8a1`)
**3 Agent 并行审计**,发现 **7 个 HIGH**:

1. **Executor ↔ Strategy pending_oids 同步断**:`_on_bar` bar-start cancel 不通知 executor → `pending_oids` 僵尸化 → `remaining=0` → **整 bar 停止发单**(灾难性 bug)
2. **BOTTOM 过期死锁**:`bottom_submitted=True` 后卡 30 分钟到 FORCE
3. **FORCE slot 基于 fill 重置**:持续成交永远 peg,cross 段从不触发
4. **TestFullModule 在 M1 bar 永不发单**:`bar_total=60s` + BOTTOM `wait<60s` → 直接回 IDLE
5. **`_peg_pending` 是 no-op**:违反核心原则 5 "挂单 bid1 追价重挂"
6. **保证金预检缺失**:plan 要求但未实现
7. **FORCE cross tick 数硬编码**:plan 本身不一致(3 vs 7-10)

**修复**(一次性全修):
- Executor 状态转换点(`on_signal`/`on_stop_triggered`/`bar_end`)返回 `list[ExecAction]` 含 cancel
- 策略层所有 `cancel_order` 同步 `register_cancelled`
- BOTTOM 过期自动 cancel+resubmit 高 urgency(只触发一次)
- FORCE slot 推进只看时间不看 fill,移除 `slot_filled_lots` 污染
- `bar_total_sec < 1200` 时 BOTTOM wait 自适应 `min(60, bar//20)`
- `pending_oids` 值从 int 变 `_PendingOrder(vol, price)` 支持真 peg
- `_peg_pending` 实际实现:target_price 漂移 ≥ 1 tick 自动 cancel+resubmit
- V8 `_on_bar` 保证金预检:`price × multiplier × vol × 0.15 > available × 0.6` 跳过
- `force_cross_min_urgency` 参数化(默认 0.75 ≈ 8 tick,可调 0.3 ≈ 3 tick)

测试 127/127 绿(+13 新测试)
